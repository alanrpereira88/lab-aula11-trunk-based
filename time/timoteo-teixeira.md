## Reflexão — Trunk-Based

- O que foi diferente em relação ao GitHub Flow?
A principal diferença foi a ausência de um Pull Request e de uma etapa de revisão de código antes da integração. No Trunk-Based, criamos uma branch de curtíssima duração (short/), fizemos o merge imediatamente na main no mesmo ciclo e até realizamos commits diretamente no tronco principal, enquanto no GitHub Flow a branch ficava isolada aguardando a aprovação do PR e a passagem do CI para ser integrada.

- Qual workflow te parece mais seguro? Por quê?
O GitHub Flow aparenta ser mais seguro para a maioria das equipes, pois a obrigatoriedade do Pull Request e do Code Review cria uma barreira colaborativa que impede códigos com erros de entrarem na branch main. Por outro lado, embora o Trunk-Based evite grandes conflitos de código ("integration hell"), ele só é realmente seguro se a equipe tiver uma maturidade técnica muito alta, dependendo de uma suíte de testes automatizados extremamente robusta e do uso de feature flags para esconder código incompleto.
