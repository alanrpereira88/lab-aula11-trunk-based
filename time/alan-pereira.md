## Reflexão — Trunk-Based

- Trunk-Based exige maturidade de testes — sem testes confiáveis, integrar na main várias vezes ao dia é arriscado.
- GitHub Flow é mais seguro para times menores ou com menos cobertura de testes.
- Usaria TBD em plataformas SaaS com deploy contínuo; GitHub Flow em projetos com ciclos de revisão mais longos.
